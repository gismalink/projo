# Workflow Checklist

Короткий чеклист на каждый инкремент.

## 1. Сделать задачу
1. Уточнить scope и критерий готовности.
2. Внести изменения в код.
3. Если меняется схема БД: добавить Prisma migration.

## 2. Локальная проверка
1. Пересобрать проект: `npm run build`.
2. Прогнать проверки: `npm run verify`.
3. Быстрый smoke-check UI/API вручную (критический сценарий задачи).

## 3. Обновить документацию
1. Обновить `README.md`, если меняются команды/флоу.
2. Обновить профильную доку в `docs/` (архитектура, roadmap, инструкции).
3. Обновить backlog/чеклист выполнения.

## 4. Завершение
1. Проверить `git status` (нет случайных файлов).
2. Сделать коммит с коротким и понятным сообщением.
3. (Опционально) Подготовить краткое описание изменений/рисков для PR.

## Что обычно забывают
1. Миграции и regeneration Prisma client после изменений схемы.
2. Проверка i18n-текстов и ошибок в `ru/en`.
3. Обновление roadmap/архитектурной заметки после крупных изменений.
4. Smoke-check после успешной сборки (сборка != корректное поведение).
